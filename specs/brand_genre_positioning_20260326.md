本文件是 RPG_Demo 的内部方向真相，不是对外 marketing 文案。

判断依据来自以下现有代码与内容真相文件：

- 模板与题材来源：`/Users/lishehao/Desktop/Project/RPG_Demo/rpg_backend/story_profiles.py`
- 产品 fallback 文案来源：`/Users/lishehao/Desktop/Project/RPG_Demo/rpg_backend/product_text.py`
- 角色矩阵来源：`/Users/lishehao/Desktop/Project/RPG_Demo/artifacts/portraits/cast_content/template_role_matrix.md`

这些文件共同指向同一个结论：

- `RPG_Demo 不是泛题材故事引擎`
- `它当前最强、最一致的方向是 editorial dossier 包装下的 civic procedural thriller`

## 定位

一句话定位：

`一款以公共记录、制度压力与城市危机为核心的互动叙事会馆。`

扩展定位：

`RPG_Demo 不是泛题材故事引擎，而是专注于“公共世界如何在压力下做决定”的互动叙事产品。`

它像什么：

- 像一座围绕档案、听证、调度、授权与问责展开的互动叙事会馆
- 像一份可以进入、推进、核验、承担后果的社论式 dossier
- 像“公共系统被迫给出答案”时的程序惊悚体验

它不像什么：

- 不像承接一切题材的通用故事沙盒
- 不像以战斗、成长、旅行或恋爱幻想为主的 RPG 容器
- 不像轻小说、爽文或泛娱乐 feed 式内容产品

核心关键词固定为：

- `档案`
- `授权`
- `配给`
- `见证`
- `秩序`

## 品牌核

当前品牌气质应明确收口为：

- `editorial`
- `dossier`
- `civic`
- `procedural`
- `restrained`
- `public consequence`

这套品牌核的含义不是“严肃感”本身，而是以下六件事始终同时成立：

- 故事不是围绕私人情绪宣泄展开，而是围绕公共系统的判断与代价展开
- 冲突不是纯抽象世界观冲突，而是有记录、程序、凭证、配额、命令链或公开后果的冲突
- UI 不是做成热闹的游戏大厅，而是做成档案册、社论页、制度文书和工作台
- 角色不是 hero poster，而是 dossier subjects，是公共结构里的承压节点
- 美术不是 glossy fantasy，而是 restrained editorial grading
- 产品情绪不是“爽”，而是“推进、核验、逼近答案、承担后果”

需要额外写死的一点是：

- `restrained` 不等于 `单调`
- `dossier subjects` 不等于 `所有角色都像同一种制度人物`
- `procedural` 不等于 `冷白办公室审美`

如果这三条不额外声明，后续执行很容易把当前方向误做成：

- 灰度过重、温度过低的统一官僚面孔
- 所有角色都只剩“严肃中年制度节点”的同质化肖像
- 只有正确感，没有地方感、材料感和观看欲望的产品表面

因此，RPG_Demo 的 UI 更像：

- 档案册
- 社论页
- 制度文书

而不是：

- 娱乐平台 UI
- 轻小说 UI
- 通用 RPG UI

## 审美执行警报

从美术与审美执行角度，当前最需要防止的不是“风格跑飞”，而是“风格执行得过于正确”。

最需要避免的三类坏结果：

- `灰化`
  - 把 restrained 执行成低饱和、低对比、低温差、低人物差异的安全灰片
- `同质化`
  - 把 dossier subject 执行成所有角色都像一个系统里长出来的人
- `行政化`
  - 把 civic procedural 执行成纯办公室、纯文书、纯制服的单场景重复

因此后续一切美术与 UI 执行都必须满足：

- 有 restraint，但仍然有视觉吸引力
- 有程序感，但仍然有材料与地方性
- 有统一 grade，但仍然有角色位阶差、职业差、环境差

换句话说，RPG_Demo 要做的是：

- `克制而不乏味`
- `制度化而不行政化`
- `统一而不单调`

## 视觉正方向

未来所有 UI 与肖像都应该更靠近以下正向感受：

- 像被翻阅过的 civic dossier
- 像带有地方纹理的制度现场
- 像一份仍有人在场、仍有代价温度的公共记录

它应当拥有的视觉特征不只包括：

- 纸面感
- 留白
- 衬线标题
- procedural label

还必须同时拥有：

- `材料感`
  - 纸、布、徽章、木桌、档案柜、灯光、窗框、石材、线路、金属件
- `地方性`
  - 港务、桥梁、议事厅、钟楼、档案室、检疫通道、听证空间这些差异必须真实存在
- `位阶感`
  - broker、witness、guardian、anchor、civic 不能长得像同一类人
- `仪式感`
  - oath、vote、certification、release、custody chain 这些动作应有仪式重量
- `微弱但真实的人情温度`
  - 产品不是热闹，也不是冷死；它需要让人感到“这里真的有人在承受后果”

## 角色美术差异化原则

后续角色与头像系统必须明确遵守：

- `anchor`
  - 更像承压的公共核心人物，通常更稳、更重、更有责任痕迹
- `guardian`
  - 更像守门、控局、执行边界的人，轮廓更硬，防御性更强
- `witness`
  - 更像掌握记录、见证、转述或证词的人，表情与手部信息更关键
- `broker`
  - 更像中介、掮客、制度交易者，必须有 leverage 感，不能总退化成 generic lead
- `civic`
  - 更像处于公共系统日常压力中的承担者，不能既无权力也无质地

必须明确反对的执行偷懒方式：

- 不要把不同角色位都做成“同一张成熟、体面、冷静的人像”后只换衣服
- 不要让所有正向变体都只等于“更亮、更笑”
- 不要让所有负向变体都只等于“更暗、更皱眉”
- 不要让所有中性变体都只剩标准证件照式正对镜头

更具体地说：

- 角色差异必须先体现在身份与轮廓上，再体现在表情上
- 表情变化必须与环境变化一起成立，而不是只做嘴角差异
- 同一 story trio 必须能在缩略图里快速读出谁掌权、谁见证、谁经手、谁承压

## 三层题材边界

### 第一层：核心题材边界

这是当前产品最强、最一致、最值得继续打磨的中心区域。

- 公共记录
  例如档案、账本、笔录、认证、证词、 custody chain、 warning bulletin
- 城市基础设施
  例如桥梁、港口、电网、配套节点、调度线路、检疫与物流瓶颈
- 配给 / 物流
  例如 ration、 allotment、 convoy、 manifest、 clearance、 shelter capacity
- 授权 / 合法性
  例如 mandate、 charter、 referendum、 oath、 compact、 emergency authority
- 公共秩序 / 黑停 / 听证
  例如 blackout、 curfew、 panic、 hearing、 public square、 crowd threshold

### 第二层：相邻可扩题材

这些方向可以扩，但前提是仍然把故事钉在公共程序与公共后果上，而不是切换到另一种情绪宇宙。

- 港口与物流
  适合继续展开为 customs、 manifest、 quarantine politics、 relief routing
- 灾害预警
  适合展开 warning record、 forecast credibility、 shelter access、 response timing
- 城市边区与通行权
  适合做 checkpoint、 corridor access、 selective passage、 district asymmetry
- 议会 / 公投 / 授权妥协
  适合做 vote legitimacy、 coalition bargains、 emergency mandate、 compact pricing
- 公共证词与制度真相
  适合做 transcript、 testimony、 omission、 release timing、 procedural denial

### 第三层：不建议优先扩展

这些并不是“永远不能做”，而是当前产品心智不匹配，不应优先投入。

- 纯恋爱
- 纯校园
- 轻喜剧
- 通用奇幻冒险
- 地下犯罪爽文
- 泛世界观沙盒

它们的问题不是题材本身低级，而是它们天然要求另一套产品承诺：

- 更强的私人关系驱动
- 更强的角色成长或幻想兑现
- 更松的公共系统约束
- 更娱乐化的 UI 与文案语言

这与 RPG_Demo 当前已经形成的 civic procedural 心智相冲突。

## 允许 / 不允许的故事类型

### 允许故事的最低准入规则

一个新故事至少要满足以下 4 条中的 2 条，才应该进入当前主产品轨道：

- 存在公共决策压力
- 存在可见制度或程序
- 存在公共后果而非纯私人后果
- 存在记录 / 证词 / 配额 / 秩序 / 授权中的至少一种

### 非常适合

- 围绕档案、账本、笔录、认证展开的真相与程序故事
- 围绕配给、桥梁、港口、检疫、物流调度展开的城市压力故事
- 围绕黑停、恐慌、宵禁、公共秩序阈值展开的秩序惊悚故事
- 围绕公投、听证、 charter、 mandate、 compact 展开的合法性故事
- 围绕证词、 release timing、 warning record、 official denial 展开的制度真相故事

### 可以做但应谨慎

- 带有人际冲突的故事，但人际冲突必须嵌在公共系统里
- 带有浪漫张力的故事，但 romance 只能是制度压力下的副线，不可成为主引擎
- 带有世界观奇观的故事，但奇观必须服务公共后果，而不是抢走程序核心
- 带有地方民俗、宗教或仪式感的故事，但必须落回 oath、 legitimacy、 public consent 这类公共语义
- 带有边区、商路、港务、治安等冒险外壳的故事，但不能滑向通用冒险跑图产品

### 当前产品心智不匹配

以下类型当前不应优先承接：

- 冲突只发生在私密关系里的故事
- 世界推进不依赖公共系统的故事
- 用户幻想主要是战斗、恋爱、成长或旅行的故事
- 以爽点兑现、 power fantasy 或角色收集为主轴的故事
- 主要依赖反派强弱、数值强度、地图探索或恋爱推进的故事

它们的共同问题是：

- 用户不是在迫使系统给出答案
- 叙事不需要记录、程序、授权、证词或问责来支撑
- 产品的 dossier 式 UI 和 restraint 语气会显得不合身

## UI 语言原则

语言原则固定为：

- 像档案，不像广告
- 像审稿，不像热闹 feed
- 像会馆记录，不像游戏大厅

视觉原则固定为：

- `paper surface`
- `restrained palette`
- `serif headline + mono label`
- `image grading is editorial, not glossy`
- `portraits are dossier subjects, not hero posters`
- `local texture matters`
- `ritual weight matters`
- `human warmth must remain traceable`

这些原则对应的实际 UI 语法应继续保持：

- 页面像 paper stack、 record panel、 tool surface，而不是 neon dashboard
- 版式优先层级、留白、标签化元数据，而不是游戏化爆点组件
- 标题承担 editorial weight，标签承担 procedural clarity
- 人像以 cover crop 进入页面，但身份呈现必须服务 dossier 可读性，而不是角色海报感
- CTA 应偏审议、推进、核验、发布、记录、继续，而不是“开打 / 冲刺 / 开箱”

并且要继续补充三条执行线：

- 页面必须像“有人处理过、翻阅过、批注过”的纸面，而不是无菌组件库
- 模块必须有轻重主次，不允许所有卡片同等响亮
- 每个页面都应保留一点“公共现场”的空气，而不是只剩规则面板

文案原则固定为：

- 少用煽动性游戏词
- 少用爽文式承诺
- 多用“记录 / 结构 / 压力 / 后果 / 证词 / 问责 / 程序”词汇
- 少把用户当“征服世界的主角”，多把用户放在“逼近答案与承担判断”的位置

但同样需要防止另一种偏差：

- 不要把文案写得只有制度词，没有人味
- 不要把 restraint 写成无情绪
- 不要让所有页面都像同一份行政通知

RPG_Demo 的语气应该是：

- 克制
- 准确
- 有判断重量
- 但仍然有叙事张力与人类温度

## 产品聚焦建议

未来主方向应明确打磨为：

`公共危机中的制度惊悚互动叙事`

这句话比“AI 故事生成器”更准确，也比“万能故事平台”更有产品辨识度。

明确反对的方向：

- 不要继续把产品往“什么故事都能接”的方向拉
- 不要为了兼容所有题材，稀释掉 dossier、 civic、 procedural 这三层共同语言
- 不要为了追求更大题材池，把 UI、文案、角色位和故事模板改造成泛娱乐平台语法

未来若继续扩题材，只允许做相邻扩展，不要跨到完全不同的情绪宇宙。判断标准是：

- 新模板是否仍然围绕公共系统承压
- 新角色是否仍然像制度节点、见证者、执行者、经手人、掮客，而不是泛主角团成员
- 新 UI 是否仍然像记录、审阅、推进与问责的空间
- 新美术是否仍然能维持地方性、材料感和角色位阶差，而不是滑回 generic prestige illustration

对后续执行的直接建议：

- 新模板优先从 archive / legitimacy / logistics / public-order / warning-record 的相邻题材里扩
- 新角色继续围绕 anchor / guardian / witness / broker / civic 这类程序角色位构建
- 新文案继续坚持 restrained editorial tone，不回到 generic game hype
- 新美术与头像继续服务 dossier subject 呈现，而不是英雄海报化
- 后续应补三份下游真相文件：
  - `visual_art_direction_guide`
  - `character_archetype_visual_rules`
  - `brand_copy_naming_guide`

结论：

RPG_Demo 最值得打磨的，不是“承接一切故事”的宽度，而是“把公共危机中的制度压力讲得更准、更稳、更有辨识度”的深度。
